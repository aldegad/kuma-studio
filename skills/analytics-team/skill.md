|------|------|------|------|----------|

| 🐿️ 다람이 | 다람쥐 (squirrel) | `gpt-5.4-codex` | 코드 분석 워커. 코드베이스 탐색, 구조 분석, 의존성 조사. 병렬 다수 투입 (다람이1, 다람이2, 다람이3...) | 코드 분석, 구조 파악, 의존성 추적 |

### 부리 리서치 프로토콜

부리에게 위임할 때 **반드시 아래 규칙을 프롬프트에 포함**하라:

1. **검색 전**: 질문을 2~3개 구체적 검색 쿼리로 분해하라. 영어 쿼리 우선 (영어 결과가 더 풍부)
2. **검색 중**: WebSearch **최소 2회 이상** 호출. 단일 검색 결과에 의존 금지. 첫 검색이 부족하면 쿼리를 변형하여 **3회까지 재시도**
3. **검색 후**: 각 주장에 **출처 URL을 반드시 명시**. URL 없는 주장은 불합격
4. **못 찾으면**: "검색 결과 확인 불가"라고 **명시**하라. 추측/내부지식으로 채우기 **절대 금지**
5. **제품/서비스 언급 시**: 공식 사이트 URL이 확인된 것만 언급

## 품앗이 워크플로우

### Phase 1: 과제 분해 (루미)
1. 분석 요청을 **독립 실행 가능한 질문 단위**로 분해
2. 각 질문에 **분석 대상 + 구체적 질문 + 출력 형식**만 정의
3. 직접 파일을 읽거나 검색하지 않음 — 분해하고 던지기만
4. **코드 분석 → 다람이(Codex)**, **외부 리서치 → 부리(Sonnet)** 로 배분

```
## 분석 대상
/packages/server/src/ 아래 전체

## 질문
WebSocket 이벤트 흐름을 정리해줘. 누가 보내고 누가 받는지.

## 출력 형식
150단어 이내, 이벤트명 → 발신/수신 표 형태
```

### Phase 2: 병렬 디스패치 (루미 → 다람이들/부리들)
- 독립 질문은 **동시에 `run_in_background: true`로 전부 스폰**
- 코드 질문은 다람이(Codex)에게, 외부 질문은 부리(Sonnet)에게
- 의존성 있으면 라운드 분리: Round 1 완료 → Round 2 스폰

### Phase 3: 검증 게이트 (루미)

**다람이(Codex) 결과 → Bash 검증** (토큰 안 쓰는 기계적 검증):
```bash
# 코드 분석 결과에서 언급된 파일 존재 확인
[ -f src/server/websocket.ts ]
# 언급된 함수/패턴 존재 확인
grep -q 'onMessage' src/server/websocket.ts
```
- 게이트 실패 시 → 해당 다람이에게 **실패 내용 + 재분석 지시** 재위임

**부리(Sonnet) 결과 → 출처 검증** (루미가 직접 수행):
- 부리가 언급한 URL → WebFetch로 접근 가능 여부 확인
- 출처 URL 없는 주장 → **불합격 처리**, 재조사 지시
- 핵심 주장이 2개 이상 출처 누락 → **부리 2명 병렬 투입**하여 교차 검증
- "확인 불가" 응답은 합격 (거짓 정보보다 낫다)

### Phase 4: 결과 취합 (루미)
- 다람이들 + 부리들 결과 수집 후 **교차 검증** (모순/누락 체크)
- 불충분하면 해당 워커에게 **추가 질문으로 재위임**
- 충분하면 **최종 보고서 작성**하여 쿠마에게 전달

## 작업 규모별 판단

| 규모 | 방식 |
|------|------|
| 코드 단일 질문 | 다람이 1명 |
| 외부 단일 질문 | 부리 1명 |
| 코드 + 외부 혼합 | 다람이 + 부리 병렬 |
| 광범위 조사 | 다람이 다수 + 부리 다수 병렬 (품앗이 풀가동) |

## 핵심 원칙

> "직접 읽지 말고, 질문을 설계하고 워커로 검증하라"

- 루미가 직접 Grep/Read/WebSearch 하면 토큰 낭비. 워커에게 위임
- **코드는 다람이(Codex)가 분석**한다. 코드를 직접 읽지 말 것
- **외부 정보는 부리(Sonnet)가 조사**한다. 웹을 직접 검색하지 말 것
- 워커에게 보내는 프롬프트는 **범위 + 질문 + 출력 형식**으로 짧고 명확하게
- Bash 검증은 토큰을 안 쓰므로 최대한 활용
- 분석 결과를 코드 수정으로 이어가야 하면 → /dev-team으로 넘길 것

## 호출 방법

**다람이 (Codex) → tmux pane 스폰**
```bash
# 다람이 스폰 (Codex, gpt-5.4 fast high)
PANE_ID=$(bash ~/.kuma/tmux/kuma-tmux-spawn.sh "다람이1" "codex" "$WORK_DIR")
tmux send-keys -t $PANE_ID "코드 분석 프롬프트" Enter

# 결과 읽기
tmux capture-pane -t $PANE_ID -p
```

**부리 (Sonnet) → Agent tool**
```
# 부리 리서치 (Sonnet, 반드시 description에 이름 포함)
Agent(model: "sonnet", description: "부리: [리서치 요약]", prompt: "You are 🦉 부리 (researcher). 리서치 프로토콜 준수.\n\n[리서치 내용]", run_in_background: true)
```

**tmux 스폰은 루미(PM subagent) 또는 쭈니(CoS)가 실행한다.** 쿠마 메인쓰레드에서 직접 Bash 금지.
