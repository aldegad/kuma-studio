# Chrome Web Store Listing Copy

## Short description

Playwright-shaped browser sharing for coding agents, with picks, job cards, and visible paw feedback in your real browser.

## Detailed description

Kuma Picker lets you share your **real browser** with a coding agent instead of pushing the work into an isolated test browser.

Pick an element or drag an area on the page, attach a job, and keep the task visibly anchored to the UI with job cards. When the agent works, Kuma Picker keeps the interaction visible with playful paw feedback so you can see where it clicked, dragged, or held.

### What Kuma Picker does

- pick an element or area directly from the page
- leave visible job cards attached to the exact UI you want changed
- run a Playwright-shaped automation subset in the browser you already have open
- keep a shared local state home so people and agents stay in sync
- show visible action feedback during browser automation

### Why it is different

- works in your real browser tabs, cookies, and session state
- supports human-agent handoff through picks and job cards
- local-first architecture with a local daemon instead of a hosted Kuma backend
- no Playwright package required for the normal Kuma workflow

### Important note

Kuma Picker talks to a **local Kuma Picker daemon** that you run on your own machine. It is intended for local development, UI debugging, and human-agent collaboration workflows.
